# 论文实验脚本说明

本文实验脚本位于 `scripts/run_paper_*.sh`。所有脚本默认从 `scripts/common.sh` 读取路径、模型和 NPU 配置。

## 运行顺序

### 1. 确定性消融

```bash
source scripts/common.sh
./scripts/run_paper_01_skill_ablation.sh
```

回答问题：topo、temporal 和融合各自贡献多少。

### 2. 信任门控离线评估

```bash
source scripts/common.sh
./scripts/run_paper_02_gate_routing.sh
```

回答问题：多少 case 可以自动接受，多少需要 LLM 或人工复核。

### 3. LLM 仲裁实验

```bash
source scripts/common.sh
./scripts/run_paper_03_llm_arbitration.sh
```

回答问题：全量 LLM 与门控 LLM 的准确率和调用成本差异。

### 4. 门控策略分析

先运行实验 3，得到 `gate_pipe_llm` 目录，然后执行：

```bash
./scripts/run_paper_04_gate_policy_analysis.sh "$PINGMESH_RESULTS/<run>/gate_pipe_llm"
```

回答问题：不同 gate policy 在同一批 LLM 结果上表现如何。

### 5. 候选摘要缓存实验

```bash
source scripts/common.sh
export PINGMESH_SUMMARY_CACHE_DIR="$PINGMESH_RESULTS/summary_cache"
./scripts/run_paper_05_precompute_summary_cache.sh
./scripts/run_paper_06_cached_summary_llm.sh
```

回答问题：小模型候选摘要是否能降低主 LLM prompt 成本。

## 结果整理建议

论文中建议至少整理这些表：

| 表 | 来源脚本 | 指标 |
| --- | --- | --- |
| 主方法对比 | `run_paper_01_skill_ablation.sh` / `run_paper_03_llm_arbitration.sh` | Top-1, Top-3, Top-5 |
| 门控路由统计 | `run_paper_02_gate_routing.sh` | route count, route coverage, route Top-K |
| LLM 成本收益 | `run_paper_03_llm_arbitration.sh` | LLM call coverage, Top-1 delta |
| 策略消融 | `run_paper_04_gate_policy_analysis.sh` | per-policy Top-K, auto accept ratio |
| 摘要缓存 | `run_paper_06_cached_summary_llm.sh` | prompt compression, Top-K delta |
